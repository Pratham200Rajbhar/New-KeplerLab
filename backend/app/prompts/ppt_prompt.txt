### SLIDE TYPE LIBRARY (pick the best for each block of content)

**1. TITLE SLIDE** (always slide 1)
- Large bold title (max 2 lines), subtitle, author/date area
- Visually striking gradient or geometric pattern background
- Centered flex column layout

**2. SECTION DIVIDER SLIDE**
- Large section number or icon, bold section title, brief description
- Strong visual contrast, used to introduce major topic groups

**3. BULLET LIST SLIDE**
- Used when: items are parallel, independent, and fit in a phrase
- Slide title + 4-8 styled bullets, each with icon/emoji + phrase
- For 7+ items: use two-column bullet layout
- Do NOT use this for content that has natural sequence or relationships

**4. NUMBERED STEPS / PROCESS SLIDE**
- Used for: procedures, how-to guides, workflows, sequences
- 3-6 numbered steps with title + 1-3 line description each
- Horizontal timeline or vertical numbered list with connector lines
- Show ALL steps — if > 5 steps, split across 2 process slides

**5. TWO-COLUMN SLIDE**
- Used for: comparisons, pros/cons, before/after, two approaches
- 50/50 or 60/40 split; each column has heading + content
- Content inside columns can itself be bullets, paragraphs, or code

**6. GRID / CARD SLIDE**
- Used for: features, components, categories, multiple independent items
- 2x2 or 3x2 grid; each card has icon + title + 1-3 line description
- Cards: rounded corners, subtle border, slight background variation

**7. STATS / METRICS SLIDE**
- Used for: numbers, performance indicators, data highlights
- 3-5 large metric boxes; each: big bold value + label + optional unit/trend
- High contrast, data-focused, minimal decoration

**8. TABLE SLIDE**
- Used for: structured comparisons, multi-attribute data, schedules, matrices
- Full HTML <table> with styled header row and alternating row colors
- Up to 8 rows, 4-5 columns; rounded table container, proper cell padding
- **Do NOT abbreviate rows** — show all data from the source

**9. CODE BLOCK SLIDE**
- Used for: code snippets, commands, config files, API examples, syntax
- Dark terminal/editor-style code area with monospace font
- Syntax color classes applied via CSS (keywords, strings, comments, functions)
- Title above code block; optional explanation below or as side column
- For long code: split into slides (Part 1 / Part 2), show all lines

**10. PARAGRAPH / EXPLANATION SLIDE**
- Used for: definitions, conceptual explanations, narrative content, descriptions
- Slide title + 2-4 paragraph blocks OR 1 large paragraph + callout card
- Use callout boxes (colored left-border card) for key terms or definitions
- Body text: 1rem-1.2rem, line-height: 1.7 — NOT compressed into bullets

**11. MIXED CONTENT SLIDE**
- Used for: topics that combine explanation + example, or text + data
- Upper half: paragraph/bullets; lower half: table, code, or metric boxes
- Or: left column = text explanation, right column = code/table/cards

**12. QUOTE / HIGHLIGHT SLIDE**
- Used for: single powerful insights, key principles, important warnings
- Large quote or key insight centered on slide, decorative quotation marks
- Minimal content — let the message breathe

**13. CLOSING / SUMMARY SLIDE** (always last slide)
- Key takeaways with 3-6 concrete bullets from the entire presentation
- Summary must reflect actual content covered, not generic platitudes
- Visually matches title slide aesthetic for bookend effect
